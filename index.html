+

Problem

권한 조회 API 응답 지연(2~5분)으로 리그레션 전체 수행 시간이 과도하게 길어지는 문제. 스텝 간 비정상적인 대기가 반복 발생.

Strategy

CDP로 API 인터셉트 → Mock 처리 → 실제 API 병렬 호출로 변경 감지 시 fixture를 자동 갱신하는 자가 업데이트 로직 적용.

Outcome

API 응답 2~5분 → 200ms 이하 (최대 1,500배). 시나리오 수행 시간 30% 단축. flaky test 제거 및 CI 환경 안정성 확보.

핵심 인사이트

테스트 속도와 Mock 최신성을 동시에 보장 — API 스펙 변경도 수동 갱신 없이 자동 반영.

자동화 프레임워크

실무에서 설계·운영한 E2E 자동화 프레임워크를 오픈소스로 공개했습니다.

⚙ GitHub에서 소스보기 —  hyunwoo-auto / selenium-cucumber-java-framework

Selenium 4 + Cucumber 7 + Java 11 기반 BDD E2E 자동화 프레임워크. PicoContainer 의존성 주입, Manager 계층 분리, CDP 기반 API 인터셉트를 적용한 실무형 구조입니다.

Java 11 Selenium 4 Cucumber 7 PicoContainer CDP Log4j2 Maven

주요 설계 결정

TestContext — PicoContainer DI Step 클래스 간 WebDriver · PageObject · Interceptor를 단일 컨텍스트로 공유. 생성자 주입으로 의존성 자동 해결.
Manager 계층 분리 WebDriverManager · PageObjectManager · FileReaderManager를 각각 분리해 단일 책임 원칙 적용. Lazy 초기화로 불필요한 인스턴스 생성 방지.
@FindBy 미사용 — By 변수 직접 선언 동적 인덱스 기반 XPath 조합이 필요한 경우 @FindBy 사용 불가. 일관성을 위해 전체를 By 변수 방식으로 통일.
CDP 기반 API 인터셉트 실제 서버 요청 없이 Mock 응답으로 즉시 대체. URL 패턴별 LinkedHashMap으로 등록 순서 보장.

시나리오 구성

login.feature — @Smoke @Login @Negative @Edge 정상 로그인 / 비밀번호 누락 / 공백 입력 등 Positive · Negative · Edge 케이스 분류.
networkIntercept.feature — @NetworkIntercept 인터셉트 전후 응답 시간 비교. httpbin.org/delay/3 (3초 지연 API)를 Mock으로 즉시 응답 처리.
태그 기반 선택적 실행 mvn test -Dcucumber.filter.tags="@Smoke"
